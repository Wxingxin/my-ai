这个文件是**新闻工具模块**，同样遵循 Function Calling 的"声明 + 执行"分离模式，但内部实现比天气工具复杂得多，核心是一套完整的 RSS 聚合管道。

---

**`newsToolDefinition`** — AI 工具声明

与 `weatherToolDefinition` 结构相同，区别是 `keyword` 参数不在 `required` 里，说明它是可选的。AI 可以在用户问"最近有什么新闻"时不传参，问"有什么 AI 新闻"时传入关键词。

---

**数据源与分类映射**

`RSS_SOURCES` 配置了 7 个 RSS 源，按 `综合 / 科技 / 财经 / 国际` 分类。`KEYWORD_CATEGORY_MAP` 则把用户可能说的关键词映射到对应分类，例如 `"ai"` / `"AI"` / `"人工智能"` 都指向 `["科技"]`，做了中英文和大小写的兼容处理。

---

**`parseItem(itemXml, sourceName)`** — 单条 RSS 条目解析器

用正则从 XML 字符串里提取字段，有几个值得注意的细节：

- 正则兼容了 `CDATA` 包裹的内容（RSS 里常见）
- `description` 会用 `.replace(/<[^>]+>/g, "")` 剔除 HTML 标签，再截取前 200 字
- `url` 优先取 `<link>`，其次降级到 `<guid>`，兼容不同 RSS 规范

---

**`fetchRss(rssUrl, sourceName)`** — 单源抓取

三个工程细节：

- `cache: "no-store"` 禁用缓存，保证每次拿最新数据
- `AbortSignal.timeout(8000)` 设置 8 秒超时，防止慢源阻塞整个请求
- 同时兼容 RSS 的 `<item>` 和 Atom 的 `<entry>` 两种格式，每源最多取 6 条

整个函数包在 `try/catch` 里，任何单源失败都静默返回空数组，不影响其他源。

---

**`getNews(keyword?)`** — 主函数，完整的聚合管道

数据流经过 5 个阶段：

```
关键词 → 匹配优先分类 → 排序数据源 → 并发拉取(前4个) → 关键词过滤 → 去重 → 截取8条
```

几个设计决策值得关注：

- **只并发前 4 个源**：用 `orderedSources.slice(0, 4)` 限制并发数，在响应速度和数据量之间取平衡
- **过滤有保底逻辑**：关键词过滤后少于 3 条时，放弃过滤结果、保留原始数据，避免返回内容太少
- **去重按标题**：用 `Set` 对标题做去重，防止多个源转载同一条新闻重复出现
- **最终最多返回 8 条**：控制传给 AI 的 tool result 体积，避免塞满上下文

---

**与天气工具的对比：**

| 维度     | 天气工具   | 新闻工具               |
| -------- | ---------- | ---------------------- |
| 数据源   | 单一 API   | 多 RSS 源聚合          |
| 容错     | 无         | 单源失败不影响整体     |
| 参数     | 必填       | 可选                   |
| 数据处理 | 直接取字段 | 解析 XML + 过滤 + 去重 |
